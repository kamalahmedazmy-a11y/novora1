# Novora — Engineering Plan to Production (خطة الإنتاج التفصيلية)

> خطة هندسية مرحلية لتحويل Novora من **MVP متقدّم** إلى **منتج SaaS قابل للبيع**.
> لكل مرحلة: المهام بالتفصيل (الملف + المكتبة + طريقة التنفيذ + معيار القبول) ·
> الميزات المضافة · حالة النظام المتوقعة عند الانتهاء · بوابة اختبار (QA Gate).

**Stack الحالي:** React 19 + Vite · Node/Express (ESM) · MongoDB/Mongoose · JWT · multi-tenant بـ `schoolId`.

**قاعدة ذهبية لكل مرحلة:** فرع جديد (`feat/phase-N`) → تنفيذ المهام → تشغيل بوابة الاختبار → دمج بعد ما كل الاختبارات تعدّي.

|---|---------|-------|---------|

# 🔴 المرحلة 0 — الأمان والأساسيات

**الهدف:** يبقى النظام آمن بما يكفي للتعامل مع بيانات أطفال حقيقية. *بدونها مفيش بيع.*

### المهام بالتفصيل

| # | المهمة | الملفات | المكتبة | طريقة التنفيذ | معيار القبول |
|---|--------|---------|---------|----------------|---------------|
| 0.1 | إزالة fallback الـ JWT | `server/routes/auth.js`, `server/middleware/authMiddleware.js` | — | استبدل `process.env.JWT_SECRET \|\| 'secret'` بـ check وقت الإقلاع: لو مش متظبط → `throw` يوقف السيرفر | السيرفر يرفض الإقلاع بدون `JWT_SECRET` |
| 0.2 | Rate limiting | `server/server.js` (route للـ auth) | `express-rate-limit` | limiter عام (100/دقيقة) + limiter صارم على `/api/auth/login` (5/دقيقة بالـ IP) | المحاولة السادسة في دقيقة → 429 |
| 0.3 | Helmet | `server/server.js` | `helmet` | `app.use(helmet())` قبل الـ routes | headers الأمان موجودة في الـ response |
| 0.4 | CORS مقيّد | `server/server.js` | `cors` (موجود) | `cors({ origin: process.env.CLIENT_ORIGINS.split(',') })` | origin غير معروف → يتمنع |
| 0.5 | Input validation | كل `server/routes/*.js` | `zod` | schema لكل body + middleware `validate(schema)` يرجّع 400 منظّم | إدخال خاطئ → 400 برسالة واضحة مش 500 |
| 0.6 | إخفاء أخطاء 500 | `server/server.js` (error handler) | — | في production: لو `err.status>=500` رجّع رسالة عامة + سجّل التفاصيل داخلياً (مش للعميل) | الـ 500 مايرجّعش stack/تفاصيل |
| 0.7 | استعادة كلمة المرور | `server/routes/auth.js` + موديل `PasswordReset` | `crypto` | `POST /forgot-password` (token عشوائي + انتهاء 30د) → `POST /reset-password` | المستخدم يقدر يغيّر باسورده عبر token صالح مرة واحدة |
| 0.8 | تحقق الإيميل | `server/models/User.js` (+`isVerified`) + auth | — | عند التسجيل ابعت token تفعيل؛ تقييد الدخول لو مش مفعّل (حسب السياسة) | حساب جديد = `isVerified:false` |
| 0.9 | Refresh tokens | `auth.js`, `authMiddleware.js` + موديل `RefreshToken` | `jsonwebtoken` | access قصير (15د) + refresh (7 أيام، rotation + إبطال) | الـ access ينتهي بعد 15د والـ refresh يجدّده |
| 0.10 | Audit log | موديل `AuditLog` + `utils/audit.js` | — | `logAudit(userId, action, entity, meta)` يتنادى عند العمليات الحساسة (تسجيل دخول، حذف، دفع، تعيين دور) | كل عملية حساسة لها سجل (مين/إيه/إمتى) |
| 0.11 | إدارة الأسرار | `.env` → secrets manager | — | نقل الأسرار خارج الكود + **تدوير الـ `JWT_SECRET` المسرّب في git history** | مفيش أسرار في الـ repo والقديم مُبطَل |

**ملاحظة حرجة:** الـ `JWT_SECRET` القديم اتكتب في الـ commits — **لازم يتدوّر (rotate)** لأنه موجود في الـ history.

### الميزات المضافة
استعادة كلمة المرور · تحقق الإيميل · refresh tokens · سجل تدقيق · حماية brute-force.

### حالة النظام المتوقعة عند الانتهاء
نظام يصمد أمام الهجمات الشائعة (brute-force، XSS headers، CORS، injection عبر validation)، بأسرار مُدارة، وقابلية تتبّع كاملة للعمليات الحساسة.

### 🧪 بوابة الاختبار (بعد المرحلة 0)
- اختبار سلبي: 6 محاولات دخول فاشلة → 429.
- إقلاع بدون `JWT_SECRET` → السيرفر يفشل بوضوح.
- إرسال body ناقص/خاطئ لـ 5 endpoints → كلها 400 منظّمة.
- محاولة CORS من origin غريب → ترفض.
- دورة كاملة: forgot → reset → دخول بالباسورد الجديد.
- مسح أمني سريع: `npm audit` + فحص headers بـ `curl -I`.
- ✅ **المعيار:** صفر تسريب أسرار، صفر 500 يكشف تفاصيل.

- ✅ **المعيار:** صفر اختبارات حمراء، صفر COLLSCAN على الـ queries الحرجة.

### حالة النظام المتوقعة عند الانتهاء
كل دور يقدر يعمل **كل** وظائفه من الواجهة بدون أي API يدوي، بعربي وإنجليزي كامل.

### 🧪 بوابة الاختبار (بعد المرحلة 2) — E2E لكل دور
- **معلم:** يعمل امتحان + واجب بمرفق → **طالب:** يشوفهم ويمتحن ويحمّل المرفق.
- **ولي أمر ↔ أدمن ↔ معلم:** رسالة بمرفق تروح وتيجي ورد.
- **مشرف:** pickup → **ولي أمر:** إشعار. **ولي أمر:** إلغاء ركوب → اختفاء من خريطة المشرف.
- تبديل اللغة على كل لوحة → صفر نص إنجليزي متبقّي.
- ✅ **المعيار:** مفيش وظيفة "شكل" — كل زر بيعمل تأثير حقيقي عبر الأدوار.
